# 외부 서비스 정보

## 1. 문서 목적

이 문서는 WISH 프로젝트에서 사용하는 외부 서비스의 용도, 필요한 계정/키, 설정 위치, 운영 시 확인해야 할 사항을 정리한 문서이다.

실제 API key(외부 서비스 인증 키), secret(비밀키), service account(서비스 계정) 원문은 이 문서에 적지 않는다. 운영 서버에서는 `infra/.env.dev`, CI/CD 변수(CI/CD 실행 시 주입되는 변수), 또는 Jenkins credential(젠킨스 비밀값 저장소)에 저장한다.

## 2. 외부 서비스 요약

| 서비스 | 사용 목적 | 주요 설정 변수 |
| --- | --- | --- |
| GMS Anthropic API | 대화 응답 생성, 주간 리포트 요약 | `GMS_KEY`, `GMS_ANTHROPIC_BASE_URL`, `GMS_ANTHROPIC_MODEL` |
| GMS OpenAI 호환 API | Whisper STT(Speech To Text, 음성 텍스트 변환) | `GMS_KEY`, `GMS_OPENAI_BASE_URL`, `WHISPER_MODEL` |
| LiveKit | 보호자/사용자 실시간 모니터링 및 영상 세션 토큰 발급 | `LIVEKIT_URL`, `LIVEKIT_API_KEY`, `LIVEKIT_API_SECRET` |
| Firebase Cloud Messaging | 보호자 앱 push notification(푸시 알림) | `FIREBASE_PUSH_ENABLED`, `FIREBASE_PROJECT_ID`, `FIREBASE_CREDENTIALS_BASE64` |
| AWS S3 | 이미지/영상 업로드 파일 저장, presigned URL 발급 | `STORAGE_TYPE`, `AWS_REGION`, `AWS_S3_PRIVATE_BUCKET` |
| YouTube Data API | 음악 검색 기능 | `YOUTUBE_API_KEY`, `YOUTUBE_BASE_URL` |
| Docker Hub | GitLab CI에서 만든 Docker image(컨테이너 실행 이미지) 저장 | `DOCKER_HUB_USER`, `DOCKER_HUB_TOKEN` |
| Jenkins | GitLab CI 이후 서버 배포 job 실행 | `JENKINS_BASE_URL`, `JENKINS_USER`, `JENKINS_TOKEN` |
| Mattermost | CI 실패/배포 결과 알림 | `MATTERMOST_WEBHOOK_URL` |
| Runpod | GPU가 필요한 AI 서버 또는 모델 실험 환경 | Runpod template, container image, HTTP port |
| Gmail SMTP | 이메일 발송이 필요한 경우의 SMTP(Simple Mail Transfer Protocol, 메일 전송 프로토콜) 후보 | Gmail 계정, 앱 비밀번호 |

## 3. GMS Anthropic API

### 3.1 사용 목적

GMS Anthropic API는 Claude 계열 LLM(Large Language Model, 대형 언어 모델)을 호출하기 위해 사용한다.

현재 코드에서 확인되는 사용 위치:

| 위치 | 용도 |
| --- | --- |
| `backend/src/main/java/com/comong/backend/domain/dialogue/config/GmsAnthropicProperties.java` | 백엔드 Claude 호출 설정 |
| `backend/src/main/java/com/comong/backend/domain/dialogue/service/ClaudeClient.java` | 백엔드에서 GMS Anthropic 호출 |
| `ai/app/services/dialogue/chat_service.py` | AI 대화 응답 생성 |
| `ai/app/services/report/summary_service.py` | 주간 리포트 요약 생성 |

### 3.2 필요한 정보

| 항목 | 설명 |
| --- | --- |
| GMS API key | GMS에서 발급받은 API 인증 키 |
| Anthropic base URL | 기본값: `https://gms.ssafy.io/gmsapi/api.anthropic.com/v1` |
| Anthropic model | 기본값: `claude-sonnet-4-5-20250929` |
| Anthropic version | 기본값: `2023-06-01` |

### 3.3 환경 변수

| 변수명 | 설명 |
| --- | --- |
| `GMS_KEY` | GMS API key |
| `GMS_ANTHROPIC_BASE_URL` | Anthropic API base URL |
| `GMS_ANTHROPIC_MODEL` | 백엔드와 리포트 요약에서 사용할 기본 Claude 모델 |
| `GMS_ANTHROPIC_VERSION` | Anthropic API version |
| `GMS_ANTHROPIC_TIMEOUT_SECONDS` | Claude 호출 timeout(응답 제한 시간) |
| `DIALOGUE_CHAT_MODEL` | AI 대화 응답용 모델 override(기본 모델을 덮어쓰는 값) |
| `REPORT_SUMMARY_MODEL` | 주간 리포트 요약용 모델 override |
| `REPORT_SUMMARY_TIMEOUT_SECONDS` | 리포트 요약용 timeout |
| `REPORT_SUMMARY_MAX_TOKENS` | 리포트 요약 응답 최대 token(모델이 처리하는 텍스트 단위) |

### 3.4 설정 위치

| 파일 | 설명 |
| --- | --- |
| `backend/src/main/resources/application.yaml` | 백엔드 `gms.anthropic` 기본 설정 |
| `infra/.env.example` | 로컬 환경 변수 예시 |
| `infra/.env.dev.example` | 개발 서버 환경 변수 예시 |

### 3.5 운영 확인

1. `GMS_KEY`가 비어 있으면 AI 대화와 리포트 요약은 fallback(외부 호출 실패 시 대체 응답)으로 동작할 수 있다.
2. 모델명이 GMS에서 허용되지 않으면 4xx HTTP 오류가 발생한다.
3. timeout이 너무 짧으면 응답 생성 도중 fallback이 자주 발생한다.
4. prompt(모델에 전달하는 지시문)와 응답 본문에 개인정보가 포함되지 않도록 운영 로그를 관리한다.

## 4. GMS OpenAI 호환 API 및 Whisper

### 4.1 사용 목적

GMS OpenAI 호환 API는 Whisper 기반 STT(Speech To Text, 음성 텍스트 변환)를 위해 사용한다. 사용자의 음성 파일을 텍스트로 변환해 대화 기능에 전달한다.

현재 코드에서 확인되는 사용 위치:

| 위치 | 용도 |
| --- | --- |
| `ai/app/services/dialogue/stt_service.py` | 오디오 파일을 Whisper API로 전송하고 텍스트 결과 수신 |

### 4.2 필요한 정보

| 항목 | 설명 |
| --- | --- |
| GMS API key | GMS에서 발급받은 API 인증 키 |
| OpenAI 호환 base URL | 기본값: `https://gms.ssafy.io/gmsapi/api.openai.com/v1` |
| Whisper model | 기본값: `whisper-1` |
| 언어 | 기본값: `ko` |

### 4.3 환경 변수

| 변수명 | 설명 |
| --- | --- |
| `GMS_KEY` | GMS API key |
| `GMS_OPENAI_BASE_URL` | GMS OpenAI 호환 API base URL |
| `WHISPER_MODEL` | STT 모델명 |
| `WHISPER_TIMEOUT_SECONDS` | Whisper API timeout |
| `WHISPER_LANGUAGE` | 인식 언어. 한국어는 `ko` |
| `WHISPER_PROMPT` | Whisper 인식 보정용 prompt |

### 4.4 운영 확인

1. `GMS_KEY`가 없으면 STT 기능은 빈 텍스트 fallback으로 처리된다.
2. 오디오 파일 크기가 너무 크면 timeout이 발생할 수 있다.
3. 운영 환경에서는 음성 데이터가 외부 API로 전송된다는 점을 개인정보 처리 기준에 맞게 관리한다.

## 5. TTS 처리 방식

현재 저장소에서 확인되는 운동 피드백 TTS(Text To Speech, 텍스트 음성 변환)는 브라우저의 Web Speech API를 사용한다.

확인 위치:

| 파일 | 설명 |
| --- | --- |
| `frontend/apps/game/src/shared/lib/feedbackTts.ts` | `window.speechSynthesis` 기반 음성 출력 |

운영 주의사항:

1. 브라우저가 Web Speech API를 지원하지 않으면 TTS가 재생되지 않는다.
2. 브라우저/OS에 설치된 한국어 음성에 따라 음색이 달라질 수 있다.
3. 참고 자료에 있는 `gpt-4o-mini-tts` 같은 외부 TTS API를 별도로 붙이려면 프론트 또는 AI 서비스에 API 호출 로직과 환경 변수를 추가해야 한다.

## 6. LiveKit

### 6.1 사용 목적

LiveKit은 실시간 영상/음성 세션 또는 보호자 모니터링 기능에 필요한 token(접속 권한 토큰)을 발급하기 위해 사용한다.

현재 코드에서 확인되는 사용 위치:

| 위치 | 용도 |
| --- | --- |
| `backend/src/main/java/com/comong/backend/domain/realtime/config/LiveKitProperties.java` | LiveKit 설정 검증 |
| `backend/src/main/java/com/comong/backend/domain/realtime/service/RealtimeTokenService.java` | LiveKit token 발급 |
| `frontend/apps/game` | 실시간 기능에서 `livekit-client` 사용 |
| `frontend/apps/guardian` | 보호자 화면에서 `livekit-client` 사용 |

### 6.2 필요한 정보

| 항목 | 설명 |
| --- | --- |
| LiveKit project | LiveKit Cloud 또는 자체 호스팅 프로젝트 |
| LiveKit URL | `wss://...` 형식의 WebSocket URL |
| API key | token 발급용 key |
| API secret | token 서명용 secret |

### 6.3 환경 변수

| 변수명 | 설명 |
| --- | --- |
| `LIVEKIT_URL` | LiveKit 서버 URL |
| `LIVEKIT_API_KEY` | LiveKit API key |
| `LIVEKIT_API_SECRET` | LiveKit API secret |

### 6.4 운영 확인

1. `LIVEKIT_URL`, `LIVEKIT_API_KEY`, `LIVEKIT_API_SECRET` 중 하나라도 비어 있으면 token 발급이 실패한다.
2. WebSocket 연결이 실패하면 Nginx proxy header와 SSL 인증서 상태를 함께 확인한다.
3. LiveKit project의 region(서비스 리전)이 사용자 위치와 멀면 지연 시간이 늘어날 수 있다.

## 7. Firebase Cloud Messaging

### 7.1 사용 목적

Firebase Cloud Messaging은 보호자에게 push notification(푸시 알림)을 보내기 위해 사용한다.

현재 코드에서 확인되는 사용 위치:

| 위치 | 용도 |
| --- | --- |
| `backend/src/main/java/com/comong/backend/domain/notification/config/FirebaseProperties.java` | Firebase 설정 검증 |
| `backend/src/main/java/com/comong/backend/domain/notification/config/FirebaseAdminConfig.java` | Firebase Admin SDK 초기화 |
| `backend/src/main/java/com/comong/backend/domain/notification/controller/GuardianDeviceTokenController.java` | 보호자 device token 저장 |

### 7.2 필요한 정보

| 항목 | 설명 |
| --- | --- |
| Firebase project ID | Firebase 프로젝트 식별자 |
| Service account JSON | Firebase Admin SDK용 서비스 계정 키 |
| Device token | 프론트에서 등록하는 기기별 푸시 토큰 |

### 7.3 환경 변수

| 변수명 | 설명 |
| --- | --- |
| `FIREBASE_PUSH_ENABLED` | 푸시 기능 활성화 여부 |
| `FIREBASE_PROJECT_ID` | Firebase 프로젝트 ID |
| `FIREBASE_CREDENTIALS_BASE64` | service account JSON을 base64로 인코딩한 값 |

### 7.4 설정 방법

1. Firebase Console에서 프로젝트를 생성한다.
2. Project settings에서 service account JSON을 발급한다.
3. JSON 원문을 base64로 인코딩한다.
4. 인코딩한 값을 `FIREBASE_CREDENTIALS_BASE64`에 설정한다.
5. 운영 서버에서 `FIREBASE_PUSH_ENABLED=true`로 활성화한다.

Windows PowerShell 예시:

```powershell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("firebase-service-account.json"))
```

Linux 예시:

```bash
base64 -w 0 firebase-service-account.json
```

### 7.5 운영 확인

1. `FIREBASE_PUSH_ENABLED=false`이면 실제 발송 대신 비활성 처리된다.
2. base64 값이 잘못되면 `Invalid FIREBASE_CREDENTIALS_BASE64 format` 오류가 발생한다.
3. service account JSON은 절대 Git에 커밋하지 않는다.

## 8. AWS S3

### 8.1 사용 목적

AWS S3는 이미지, 영상, 썸네일 같은 업로드 파일을 저장하는 데 사용한다. 백엔드는 presigned URL(정해진 시간 동안만 유효한 임시 업로드/다운로드 URL)을 발급한다.

현재 코드에서 확인되는 사용 위치:

| 위치 | 용도 |
| --- | --- |
| `backend/src/main/java/com/comong/backend/global/config/S3ClientConfig.java` | S3 client 설정 |
| `backend/src/main/java/com/comong/backend/global/storage/StorageProperties.java` | 저장소 설정 |
| `backend/src/main/java/com/comong/backend/domain/upload/service/PresignedUploadService.java` | presigned URL 발급 |
| `infra/aws-s3-bucket-setup-guide.md` | S3 버킷 구성 상세 가이드 |

### 8.2 필요한 정보

| 항목 | 설명 |
| --- | --- |
| AWS account | S3 bucket과 IAM user를 관리할 AWS 계정 |
| S3 bucket | private 업로드 파일 저장용 bucket |
| IAM user 또는 role | S3 접근 권한 |
| Access key | IAM user 방식일 때 필요한 access key |
| Secret access key | IAM user 방식일 때 필요한 secret key |

### 8.3 환경 변수

| 변수명 | 설명 |
| --- | --- |
| `STORAGE_TYPE` | `s3`로 설정하면 S3 저장소 사용 |
| `AWS_REGION` | AWS region. 예: `ap-northeast-2` |
| `AWS_S3_PRIVATE_BUCKET` | private bucket 이름 |
| `AWS_S3_PRIVATE_PREFIX` | 환경별 객체 prefix |
| `AWS_S3_PRESIGNED_TTL_SECONDS` | presigned URL 유효 시간 |
| `AWS_ACCESS_KEY_ID` | IAM user access key ID |
| `AWS_SECRET_ACCESS_KEY` | IAM user secret access key |

### 8.4 운영 확인

1. 운영 서버에 EC2 role(EC2에 부여하는 AWS 권한)을 붙일 수 있으면 access key보다 role 방식이 안전하다.
2. 현재 프로젝트는 SDK default credential chain(환경 변수, role 등 순서대로 자격 증명을 찾는 방식)을 사용한다.
3. bucket 이름, region, IAM 권한이 맞지 않으면 presigned URL 발급 또는 S3 업로드가 실패한다.
4. S3 설정 상세는 `infra/aws-s3-bucket-setup-guide.md`를 우선 참고한다.

## 9. YouTube Data API

### 9.1 사용 목적

YouTube Data API는 음악 검색 기능에서 사용한다. API key는 백엔드에만 두고, 프론트엔드에 직접 노출하지 않는다.

현재 코드에서 확인되는 사용 위치:

| 위치 | 용도 |
| --- | --- |
| `backend/src/main/java/com/comong/backend/domain/music/config/YoutubeProperties.java` | YouTube API 설정 |
| `backend/src/main/java/com/comong/backend/domain/music/service/YoutubeSearchService.java` | YouTube 검색 호출 |
| `frontend/apps/game/src/scenes/themes/music/search/YouTubeSearchScene.ts` | 백엔드 검색 API 호출 |

### 9.2 필요한 정보

| 항목 | 설명 |
| --- | --- |
| Google Cloud project | YouTube Data API v3를 활성화할 프로젝트 |
| API key | 백엔드에서 YouTube API 호출 시 사용하는 key |

### 9.3 환경 변수

| 변수명 | 설명 |
| --- | --- |
| `YOUTUBE_API_KEY` | YouTube Data API key |
| `YOUTUBE_BASE_URL` | 기본값: `https://www.googleapis.com/youtube/v3` |
| `YOUTUBE_TIMEOUT_SECONDS` | API timeout |

### 9.4 운영 확인

1. `YOUTUBE_API_KEY`가 비어 있으면 음악 검색 기능이 비활성 또는 실패 상태가 될 수 있다.
2. YouTube Data API는 quota(호출량 제한)가 있으므로 운영 중 사용량을 확인한다.
3. API key는 HTTP referrer(브라우저 호출 제한)보다 server key 기준으로 제한하는 것이 적절하다.

## 10. Docker Hub

### 10.1 사용 목적

GitLab CI에서 빌드한 Docker image를 저장하고, Jenkins 배포 job이 해당 이미지를 서버에서 pull(이미지 다운로드)하기 위해 사용한다.

현재 CI에서 사용하는 이미지 예시:

| 이미지 | 용도 |
| --- | --- |
| `imgunbread/e103-wish:backend-develop-latest` | 백엔드 dev 배포 |
| `imgunbread/e103-wish:ai-develop-latest` | AI dev 배포 |
| `imgunbread/e103-wish:frontend-game-develop-latest` | 게임 프론트 dev 배포 |
| `imgunbread/e103-wish:frontend-admin-develop-latest` | 관리자 프론트 dev 배포 |
| `imgunbread/e103-wish:frontend-guardian-develop-latest` | 보호자 프론트 dev 배포 |

### 10.2 필요한 정보

| 항목 | 설명 |
| --- | --- |
| Docker Hub 계정 | 이미지를 push할 계정 |
| Access token | GitLab CI에서 Docker Hub 로그인에 사용할 token |
| Repository | `imgunbread/e103-wish` 같은 이미지 저장소 |

### 10.3 CI/CD 변수

| 변수명 | 설명 |
| --- | --- |
| `DOCKER_HUB_USER` | Docker Hub 사용자명 |
| `DOCKER_HUB_TOKEN` | Docker Hub access token |

### 10.4 운영 확인

1. Docker Hub token 만료 또는 권한 부족 시 Kaniko image build job이 실패한다.
2. Docker Hub pull rate limit(이미지 다운로드 제한)에 걸릴 수 있으므로 운영 서버 pull 실패 로그를 확인한다.
3. 이미지 태그가 `infra/docker-compose.dev.yml`의 `*_DEV_IMAGE`와 맞아야 한다.

## 11. Jenkins

### 11.1 사용 목적

Jenkins는 GitLab CI에서 image build가 끝난 뒤 실제 서버 배포 job을 실행하는 데 사용한다.

현재 확인되는 사용 위치:

| 위치 | 용도 |
| --- | --- |
| `.gitlab-ci.yml` | Jenkins buildWithParameters API 호출 |
| `infra/docker-compose.platform.yml` | Jenkins 컨테이너 실행 |

### 11.2 필요한 정보

### 11.4 운영 확인

## 13. Runpod

### 13.1 사용 목적

Runpod은 GPU가 필요한 AI 서버 또는 모델 실험 환경을 임시로 운영할 때 사용한다. 참고 자료 기준으로 Docker image를 빌드해 Docker Hub에 올린 뒤, Runpod template에 해당 image를 등록하는 방식이다.

### 13.2 필요한 정보

| 항목 | 설명 |
| --- | --- |
| Runpod 계정 | GPU pod 생성 권한 |
| Docker image | Runpod에서 실행할 AI 서버 이미지 |
| Template | container image, disk size, port 등을 묶은 실행 템플릿 |
| Network volume | 모델 파일이나 대용량 데이터를 유지할 저장소 |
| HTTP port | FastAPI 접근 포트. 기본 `8001` |

### 13.3 기본 절차

1. AI Docker image를 빌드한다.
2. Docker Hub에 image를 push한다.
3. Runpod에서 network volume(네트워크 저장소)을 만든다.
4. Runpod template에 Docker image와 HTTP port를 등록한다.
5. Pods 메뉴에서 template으로 pod를 배포한다.
6. 생성된 HTTP endpoint를 프론트엔드 또는 백엔드 환경 변수에 반영한다.

예시:

```bash
cd ai
docker build --no-cache -t <dockerhub-account>/voida-fastapi:v1 .
docker push <dockerhub-account>/voida-fastapi:v1
```

### 13.4 운영 확인

1. Container disk는 Docker image와 모델 파일 크기에 맞게 설정한다.
2. HTTP port를 열지 않으면 외부에서 FastAPI에 접근할 수 없다.
3. GPU pod는 비용이 계속 발생하므로 사용하지 않을 때 중지한다.
4. public endpoint를 사용할 경우 인증/접근 제한을 별도로 고려한다.

## 14. Gmail SMTP

### 14.1 사용 목적

참고 자료에는 Gmail SMTP가 외부 서비스로 포함되어 있다. 다만 현재 저장소 설정 파일에서 Spring Mail 관련 운영 환경 변수는 확인되지 않는다. 이메일 인증이나 알림 메일 기능을 운영에 붙일 경우 Gmail SMTP 또는 별도 메일 서비스를 사용할 수 있다.

### 14.2 필요한 정보

| 항목 | 설명 |
| --- | --- |
| Gmail 계정 | 발신자 계정 |
| 앱 비밀번호 | 2단계 인증 사용 계정에서 생성하는 SMTP 전용 비밀번호 |
| SMTP host | `smtp.gmail.com` |
| SMTP port | `587` |
| TLS | 활성화 필요 |

### 14.3 Spring Boot 설정 예시

현재 코드에 메일 발송 기능을 추가할 경우 아래와 같은 변수를 사용할 수 있다.

| 변수명 | 설명 |
| --- | --- |
| `MAIL_HOST` | SMTP host |
| `MAIL_PORT` | SMTP port |
| `MAIL_USERNAME` | Gmail 계정 |
| `MAIL_PASSWORD` | Gmail 앱 비밀번호 |

예시 설정:

```yaml
spring:
  mail:
    host: ${MAIL_HOST:smtp.gmail.com}
    port: ${MAIL_PORT:587}
    username: ${MAIL_USERNAME:}
    password: ${MAIL_PASSWORD:}
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
```

### 14.4 운영 확인

1. Gmail 일반 비밀번호가 아니라 앱 비밀번호를 사용한다.
2. 대량 메일 발송에는 Gmail 제한이 있으므로 운영 트래픽이 많으면 전문 메일 서비스를 검토한다.
3. 현재 저장소 기준으로는 메일 발송 기능이 필수 외부 의존성인지 추가 확인이 필요하다.

## 15. 외부 서비스 비밀값 관리 원칙

1. API key, token, secret, service account JSON은 Git에 커밋하지 않는다.
2. 예시 파일에는 placeholder(대체 예시값)만 둔다.
3. 운영 서버의 실제 값은 `infra/.env.dev` 또는 Jenkins/GitLab CI 변수에만 저장한다.
4. 로그에 인증 헤더, API key, service account 본문이 출력되지 않게 한다.
5. 키를 교체하면 Jenkins, GitLab CI, 서버 `.env.dev` 값을 함께 갱신한다.
