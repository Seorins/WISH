# WISH 시연 시나리오

## 1. 시연 개요

이 문서는 WISH 서비스의 주요 기능을 시연 순서에 맞춰 정리한 문서입니다.

시연은 세 가지 관점으로 나누어 진행합니다.

1. 게임 사용자 시연
2. 보호자 웹 시연
3. 운영자 콘솔 시연

각 단계는 화면, 실행 동작, 클릭 위치, 설명 멘트 중심으로 작성했습니다.

## 2. 시연 준비

### 2.1 실행 대상

| 구분 | 시연 대상 | 설명 |
| --- | --- | --- |
| 게임 사용자 | WISH 게임 클라이언트 | 아이가 콘텐츠를 체험하는 화면 |
| 보호자 | WISH 보호자 웹 | 아이의 활동, 감정, 리포트, 실시간 화면을 확인하는 화면 |
| 운영자 | WISH Admin | 사용자, 콘텐츠, 모션 데이터를 운영자가 관리하는 화면 |

### 2.2 시연 전 확인 사항

- 게임 클라이언트가 정상 실행되는지 확인합니다.
- 보호자 웹과 운영자 콘솔에 로그인 가능한 계정을 준비합니다.
- 실시간 모니터링 시연을 위해 게임 사용자와 보호자 계정이 서로 연결되어 있어야 합니다.
- 카메라 기반 콘텐츠는 브라우저 또는 앱에서 카메라 권한을 허용해야 합니다.

## 3. 게임 사용자 시연

### 3.1 게임 시작

![게임 시작 화면](demo-scenario-assets/01-game-start.png)

- 실행: 시작 화면에서 `게임 시작` 버튼을 클릭합니다.
- 클릭 위치: 화면 중앙의 `게임 시작` 버튼입니다.
- 설명: 사용자는 WISH 게임에 접속해 첫 화면에서 게임을 시작할 수 있습니다.

### 3.2 로그인

![게임 로그인 화면](demo-scenario-assets/02-game-login.png)

- 실행: 이메일과 비밀번호를 입력한 뒤 `로그인` 버튼을 클릭합니다.
- 클릭 위치: 로그인 모달 하단의 `로그인` 버튼입니다.
- 설명: 로그인 후 사용자의 캐릭터와 활동 기록을 불러옵니다.

### 3.3 조작법 안내 확인

![조작법 안내 화면](demo-scenario-assets/03-game-control-guide.png)

- 실행: 마을에 진입하면 조작법 안내를 확인합니다.
- 클릭 위치: 안내창의 확인 또는 닫기 버튼입니다.
- 설명: 이동, 상호작용, 설정, 이모티콘 단축키를 안내합니다.

### 3.4 설정 변경

![설정 화면](demo-scenario-assets/04-game-settings.png)

- 실행: `ESC` 또는 설정 버튼으로 설정창을 열고 닉네임, 소리, 이동 속도를 확인합니다.
- 클릭 위치: 설정창 내 BGM, 효과음, 이동 속도 조절 영역입니다.
- 설명: 사용자는 플레이 환경을 직접 조정할 수 있습니다.

### 3.5 복장 변경

![복장 변경 화면](demo-scenario-assets/05-game-costume.png)

- 실행: 설정창의 복장 변경 메뉴에서 원하는 캐릭터 의상을 선택합니다.
- 클릭 위치: 캐릭터 카드 또는 성별 탭입니다.
- 설명: 사용자는 캐릭터 외형을 바꿔 개인화된 경험을 할 수 있습니다.

### 3.6 마을 이동 및 이모티콘 사용

![마을 이모티콘 화면](demo-scenario-assets/06-game-village-emote.png)

- 실행: 마을에서 캐릭터를 이동시키고 하단 이모티콘을 선택합니다.
- 클릭 위치: 화면 하단의 이모티콘 버튼입니다.
- 설명: 마을에서는 건물 이동뿐 아니라 간단한 감정 표현도 사용할 수 있습니다.

### 3.7 NPC 대화

- 설명: 사용자는 NPC와 대화하면서 현재 기분이나 선호를 표현할 수 있습니다.

## 6. 시연 마무리 멘트

WISH는 아이가 게임처럼 콘텐츠를 체험하는 화면, 보호자가 활동과 감정을 확인하는 화면, 운영자가 서비스 데이터를 관리하는 화면으로 구성되어 있습니다.

게임 사용자는 음악, 미술, 체조, 태권도, 대화 콘텐츠를 통해 활동 데이터를 만들고, 보호자는 이 데이터를 바탕으로 아이의 상태를 확인하며 별빛 에너지를 보낼 수 있습니다. 운영자는 전체 사용자와 콘텐츠 데이터를 관리해 서비스 운영 상태를 확인합니다.
